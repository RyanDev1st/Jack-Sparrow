# Skill: Avant-Garde Tech Web Design Synthesis

## Description
This skill dictates the front-end design language and visual architecture for website generation. It synthesizes high-performance tech utility with avant-garde, surrealist boldness. The goal is to create interfaces where brutalist/pop-art visuals meet hyper-refined, liquid-smooth digital environments.

## Core Design Philosophy
The UI must feel like a precision tool wrapped in an art exhibition. It requires the seamless, dark-mode fluidity and keyboard-centric focus of modern productivity apps, intersected with the bold, rule-breaking layouts and vibrant focal points of editorial design. 

## Instructions & Descriptive Guidelines

### 1. Layout & Spatial Architecture
* **Fluid Modularity:** Architect the layout using component-based structures that feel cohesive but can stand completely alone. 
* **Asymmetric Harmony:** Embrace asymmetry and unconventional positioning for hero sections and feature highlights. Channel editorial and artistic catalogs.
* **Anchored Utility:** While the visual presentation should be bold, core navigation, command palettes, and utility interactions must remain strictly anchored, predictable, and highly accessible.
* **Z-Axis Depth:** Create a sense of depth using layered elements, overlapping typography, and floating components. Avoid flat, single-plane designs.

### 2. Lighting, Color & Atmosphere
* **The "Liquid Aurora" Effect:** Utilize deep, dark base environments to make foreground elements ignite. Employ soft, glowing gradients, "liquid glass," or "aurora" lighting effects behind translucent panels to create an ethereal, high-tech atmosphere.
* **Surreal Accents:** Inject bold, high-saturation, contrasting accent colors (e.g., vibrant reds, electric greens, neon pinks) exclusively into interactive triggers, key metrics, or focal graphic elements to create a "pop-art" shock value.
* **Dynamic Theming:** Define color relationships dynamically through contrast ratios and atmospheric descriptions. Rely on CSS variables and theme engines for the final output.

### 3. Typography as Structure
* **High Contrast Pairing:** Contrast massive, grotesque, or editorial display fonts for headings against highly legible, geometric sans-serifs for utility text and data.
* **Graphic Text:** Treat typography as a structural graphic element, not merely a vessel for reading. Allow text to overlap containers, interact with imagery, and define spatial boundaries.

### 4. Micro-Interactions & Materiality
* **Glassmorphism:** Use frosted glass effects on cards, modals, and dropdowns to maintain background context while ensuring absolute foreground legibility.
* **Tactile Responsiveness:** Transitions must be fluid, snappy, and responsive. Buttons and cards must react with tactile feedback—subtle scaling, border illuminations, or shifting glow states on hover/focus.

## 🚫 STRICT ANTI-PATTERNS (CRITICAL INSTRUCTIONS)
* **NO ABSOLUTE SPECIFICATIONS:** Never output specific technical specs for sizing. Do not command the system to use absolute pixel values (e.g., `width: 400px` or `margin: 15px`). You must use descriptive sizing and responsive units (`rem`, `vh`, `vw`, `%`, `fr`).
* **NO HARDCODED COLORS:** Never instruct the use of specific background color codes or hex values in the overarching logic. Describe the visual intent, contrast requirements, and vibe instead.
* **NO BOILERPLATE CORPORATE SAAS:** If the resulting structure looks like a standard, flat Bootstrap template, it is a failure. Avoid generic, sanitized layouts.
